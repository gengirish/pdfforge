## Main post

We just shipped **PDFforge v2**.

If you’ve ever chained five separate calls to merge PDFs, encrypt the result, and hope nothing times out mid-run—this release is for you.

**What’s new**

→ **Pipeline API** — multiple PDF steps in one HTTP request (e.g. merge then encrypt).

→ **Batch API** — the same operation across many files, one structured response.

→ **Async jobs + optional webhooks** — queue work, poll job status, or get pushed when it’s done.

→ **OpenAPI, Swagger UI, ReDoc** — shipped with the server, not as an afterthought.

→ **Python SDK** + **MCP** — script it or plug it into agent workflows (e.g. Claude).

The browser experience is unchanged at heart: merge, split, rotate, extract text, encrypt, and decrypt—with a privacy-first, self-hostable story. v2 adds the **automation layer** when you’re ready to go beyond the upload button.

Try it: https://pdfforge.intelliforge.tech  
API docs (Swagger): https://pdfforge-api.fly.dev/api/v1/docs

Built by IntelliForge AI. Open source core; details on GitHub from the site.
